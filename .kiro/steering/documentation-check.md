# Documentation Standards

## Always Check Documentation

Before making changes or answering questions, review relevant documentation:

1. **Project Documentation**
   - Check `README.md` for project overview and setup instructions
   - Review `DEPLOYMENT.md` for deployment details and API endpoints
   - Consult `backend/docs/` for API documentation

2. **Code Documentation**
   - Read existing code comments and docstrings
   - Understand the current implementation before suggesting changes
   - Maintain consistency with existing patterns

3. **Steering Files**
   - Review `.kiro/steering/` files for project-specific guidelines
   - Follow established conventions (API standards, development guidelines)

## When to Update Documentation

Update documentation whenever you:
- Add new features or endpoints
- Change API behavior or response formats
- Modify deployment procedures
- Update dependencies or requirements
- Add new configuration options

## Documentation Locations

- **README.md**: Project overview, setup, usage examples
- **DEPLOYMENT.md**: Deployment details, API endpoints, testing
- **backend/docs/**: API reference documentation
- **Code comments**: Inline documentation for complex logic
- **.kiro/steering/**: Development guidelines and standards
