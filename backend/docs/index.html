Events API Documentation

Base URL: https://ze500yz351.execute-api.us-west-2.amazonaws.com/prod/

Overview

A RESTful API for managing events with full CRUD operations. Built with FastAPI and deployed on AWS Lambda.

Event Model

Field Type Required Validation
eventId string No Auto-generated if not provided
title string Yes 1-200 characters
description string Yes 1-1000 characters
date string Yes YYYY-MM-DD format
location string Yes 1-200 characters
capacity integer Yes Greater than 0
organizer string Yes 1-200 characters
status string Yes active, inactive, cancelled, or completed

Endpoints

GET /events

List all events with optional filtering.

Query Parameters

  • status (optional): Filter events by status (active, inactive, cancelled, completed)

Response

Status: 200 OK
Content-Type: application/json

[
  {
    "eventId": "123",
    "title": "Tech Conference",
    "description": "Annual tech conference",
    "date": "2024-12-15",
    "location": "San Francisco",
    "capacity": 500,
    "organizer": "Tech Corp",
    "status": "active"
  }
]

POST /events

Create a new event.

Request Body

{
  "title": "Tech Conference",
  "description": "Annual tech conference",
  "date": "2024-12-15",
  "location": "San Francisco",
  "capacity": 500,
  "organizer": "Tech Corp",
  "status": "active"
}

Response

Status: 201 Created
Content-Type: application/json

{
  "eventId": "generated-uuid",
  "title": "Tech Conference",
  "description": "Annual tech conference",
  "date": "2024-12-15",
  "location": "San Francisco",
  "capacity": 500,
  "organizer": "Tech Corp",
  "status": "active"
}

GET /events/{eventId}

Get a specific event by ID.

Path Parameters

  • eventId: The unique identifier of the event

Response

Status: 200 OK
Content-Type: application/json

{
  "eventId": "123",
  "title": "Tech Conference",
  "description": "Annual tech conference",
  "date": "2024-12-15",
  "location": "San Francisco",
  "capacity": 500,
  "organizer": "Tech Corp",
  "status": "active"
}

Error Response

Status: 404 Not Found
Content-Type: application/json

{
  "detail": "Event not found"
}

PUT /events/{eventId}

Update an existing event. Supports partial updates.

Path Parameters

  • eventId: The unique identifier of the event

Request Body

{
  "title": "Updated Title",
  "capacity": 600
}

Response

Status: 200 OK
Content-Type: application/json

{
  "eventId": "123",
  "title": "Updated Title",
  "description": "Annual tech conference",
  "date": "2024-12-15",
  "location": "San Francisco",
  "capacity": 600,
  "organizer": "Tech Corp",
  "status": "active"
}

DELETE /events/{eventId}

Delete an event.

Path Parameters

  • eventId: The unique identifier of the event

Response

Status: 200 OK
Content-Type: application/json

{
  "message": "Event deleted successfully"
}

Error Handling

The API returns appropriate HTTP status codes and error messages:

  • 400 Bad Request: Invalid input or validation error
  • 404 Not Found: Resource not found
  • 500 Internal Server Error: Server error

CORS

CORS is enabled for all origins, allowing web applications to access the API.
